# Models & Pricing

**The same models as Anthropic and OpenAI direct.** The difference: pay per
token with QRIS. No USD card, no monthly subscription, no FX card fee.

## Display name vs Model ID

Names such as **GPT-5.6 Sol** are human-readable **display names**. When calling
the API, use the exact **Model ID**, such as `gpt-5.6-sol`. Model IDs are
**case-sensitive**: do not change capitalization or pass the display name as
the `model` value.

List every Model ID directly from the live catalog:

```bash
curl -s https://api.nexotao.com/models | jq -r '.models[].model'
```

Example mappings:

| Display name | API Model ID |
| --- | --- |
| GPT-5.6 Luna | `gpt-5.6-luna` |
| GPT-5.6 Terra | `gpt-5.6-terra` |
| GPT-5.6 Sol | `gpt-5.6-sol` |

## Headline per 100M tokens

Round marketing figures for order-of-magnitude planning. Computed from a
blended 30% input / 70% output mix — close to what a real product workload
looks like. The actual bill is still counted per token; see the live table
below.

| Model              | Headline per 100M tokens | Positioning                                                  |
| ------------------ | ------------------------ | ------------------------------------------------------------ |
| GPT-5.6 Sol        | **Rp 700 thousand**      | Top GPT-5.6 tier; the same rate for input and output. |
| GPT-5.6 Terra      | **Rp 765 thousand**      | Balanced for coding, analysis, and agent workflows. |
| GPT-5.6 Luna       | **Rp 399 thousand**      | Efficient for everyday work and high-volume tasks. |
| Claude Opus 4.8    | **Rp 1 million**         | Anthropic's strongest reasoning model. **Far cheaper** than Anthropic list. |
| Claude Sonnet 4.6  | **Rp 700 thousand**      | Fast + smart for high-volume product chat. |
| gpt-5-mini         | **Rp 399 thousand**      | OpenAI's cheapest tier — great for classifiers, extract, batch. |
| Claude Haiku 4.5   | *coming*                 | Anthropic's fastest and cheapest. Pending catalog activation. |

"vs Anthropic list" is deliberately shown as a multiplier, not an absolute
figure, so the copy stays accurate when Anthropic changes their own list
price.

> **Match the model to the job.** `DeepSeek` is great for **high-volume, cheap**
> workloads — classification, extraction, short chat — but **not for
> agentic/coding or large context**. Its capacity is small (~20k tokens/min), so
> a single agentic or large-context request can hit `429 server at capacity`.
> For **coding/agentic & large context**, use **Claude (Opus/Sonnet)** or
> **GPT-5.6**.

## Live per-million-token rates

Beyond the headline above, every model has per-million-token rates for
**input**, **output**, and (for Claude/GPT) **cache-read + cache-write**. The
values here come straight from `GET /models` — always in sync with the live
catalog, with no hardcoded prices on the page.

## How to read the rates

- **Text** — rate per 1 million tokens, separately for input and output.
- **Cache (Claude/GPT)** — cache reads are much cheaper than the standard input
  rate; cache writes are slightly more. Full breakdown in [Billing & Pricing](/penagihan).
- **Headline per 100M tokens** — a planning figure, not the billed amount.
  The bill is always `token_count × model_rate_at_the_time`.

## Why "per 100M tokens"?

Modern text models measure real project usage in millions of tokens. A
`per 100M tokens` figure is the right order of magnitude for comparing
costs across models quickly, without getting distracted by decimal points
on the per-million-token rate.

Billing details, cache mechanics, and per-request accounting live in
[Billing & Pricing](/penagihan).
