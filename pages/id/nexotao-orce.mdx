# Nexotao Orce

**Claude Code, di browser — plus sistem orkestrasi multi-agent.** Orce kamu jalankan
sendiri di sebuah VM, lalu kendalikan dari perangkat mana pun: ngobrol dengan satu agent,
atau beri sebuah goal dan biarkan **planner memecahnya jadi task DAG** yang dikerjakan tim
agent secara paralel.

> Kalau **Claude Code** itu satu agent di terminal-mu, **Orce** adalah satu ruangan penuh
> agent — terencana, terlacak, dan ada batas biayanya, dari satu web console yang rapi.

Kode & rilis: [github.com/rizkyriyadi/nexotao-orchestrator](https://github.com/rizkyriyadi/nexotao-orchestrator)
· paket npm: [`nexotao-orce`](https://www.npmjs.com/package/nexotao-orce).

---

## Kenapa Orce

Kamu suka Claude Code, tapi terikat ke terminal di mesin yang harus terus menyala. Orce
menaruh engine yang sama (**Claude Agent SDK** resmi) di balik web UI yang kamu host
sendiri — jadi bisa nge-prompt dari HP, laptop, atau tablet — plus lapisan orkestrasi untuk
kerjaan yang terlalu besar buat satu agent.

- 💬 **Chat** — Claude Code penuh: bash, edit, read, web, subagent. Streaming, sesi
  tersimpan yang bisa dilanjutkan dari perangkat mana pun.
- 🧠 **Orce (orkestrasi)** — beri goal; **planner** memecahnya jadi **task DAG**, menugaskan
  tiap task ke agent terbaik, lalu scheduler menjalankannya **paralel selama dependency
  mengizinkan**.
- 🎫 **Sistem task** — tiap task jadi tiket (`TASK-0001`) dengan status, dependency, biaya,
  dan halaman detail.
- ✅ **Approval rencana** — tinjau DAG usulan lalu **Approve & run**, atau biarkan auto-run.
- 🤖 **Custom agent** — atur peran, system prompt, model, tool, dan workspace terisolasi.
- 💰 **Budget hard-stop** — pasang batas USD per run; warning di 80%, berhenti di 100%.

## Pasang

Butuh **Node.js 20+**. Provider model diatur di dalam aplikasi (lihat bawah).

### npm (disarankan)

```bash
npm install -g nexotao-orce
nexotao
```

Ketik **`nexotao`** untuk menjalankan server dan **membuka UI di browser**. Saat pertama
jalan kamu **membuat password sendiri di aplikasi** (tidak ada yang dicetak ke terminal),
lalu onboarding memandu proyek pertamamu.

- Berhenti: tekan **`Ctrl+C`**.
- Lupa password? jalankan `nexotao reset-password`.
- Port lain: `nexotao --port 9000` · tanpa auto-open: `nexotao --no-open`
- Tanpa install global: `npx nexotao-orce`.

### Docker

```bash
git clone https://github.com/rizkyriyadi/nexotao-orchestrator.git && cd nexotao-orchestrator
cp .env.example .env       # set APP_PASSWORD, (opsional) DATABASE_URL
docker compose up -d --build
# → http://localhost:8787
```

## Sambungkan ke Nexotao

Saat pertama jalan — dan kapan pun di **Settings** — kamu pilih sumber akses model. Pilih
**Nexotao** biar bayar per token pakai saldo Rupiah:

1. Tempel API key `sk-nexo-...`.
2. Pilih model — Claude (`claude-opus-4-8`, `claude-sonnet-4-6`), GPT, Grok, atau DeepSeek.

Alternatif **Claude Code (local)** menyambung langsung ke `claude` yang sudah login di mesin
itu (langganan, `ANTHROPIC_API_KEY`, atau Bedrock/Vertex/Foundry) — tidak ada yang
ditempel. Provider bisa diganti kapan saja; pilihannya global untuk instalasimu.

> **Belum punya key?** Daftar di [nexotao.com](https://nexotao.com) (cukup email), top-up
> QRIS mulai Rp 10.000, lalu buat key `sk-nexo-...` di menu **API Keys**. Detail di
> [Quickstart](/quickstart) dan [Autentikasi](/autentikasi).

## Dua mode

| Mode | Isi |
| ---- | --- |
| **Chat** | Satu agent, toolset penuh — bash, edit, subagent, web. |
| **Orchestrate** | Agent paralel per-lens, direkonsiliasi oleh synthesizer. Planner → task DAG → scheduler paralel, dengan biaya & budget. |

## Keamanan

Server ini bisa **menjalankan perintah shell dan mengedit file** di host-nya. Perlakukan
seperti SSH:

- Selalu taruh di belakang **HTTPS** (mis. Caddy: `reverse_proxy localhost:8787`). Jangan
  ekspos port mentah.
- Pakai **password kuat** (kamu set saat pertama jalan). Single-operator — jangan dibagikan.
- Default-nya `bypassPermissions` (agent jalan tanpa prompt). Set
  `PERMISSION_MODE=acceptEdits` untuk lebih ketat, dan kunci firewall VM ke IP-mu.
